how it works

How it works.

Three things, and that's the whole story.

// reads your % locally

Pulls the actual context-window percentage straight from Claude Code's status-line data, on your machine. No guessing, no transcript scraping, no network call.

// zero tokens

It reads a number Claude Code already computed — it never calls the model. Runs locally on every render. It will never cost you a cent or a single token of context.

// never crashes

Every failure path falls back to safe output and exits clean. It will never garble your status line. (We have tests. 52 of them.)

why it works

Why a fill meter tells you anything.

It's not a gimmick — it's measuring the actual cause of the dumbness.

Context rot. As the context window fills, the model has more and more to hold in its head — and it starts losing track of earlier details: the function it wrote, the bug you already fixed, the convention you agreed on twenty messages ago. Quality degrades before you notice it in the output.

You can't feel that coming. But the fill level is the warning sign — it climbs in plain sight, and it climbs before the answers go bad. dumbometer just puts that number where you'll actually see it, so /compact stops being a guess.
